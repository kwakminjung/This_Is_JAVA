<h1>중첩 클래스</h1>
<h2> : 클래스 내부에 선언한 클래스를 말함 </h2>
<br>
클래스가 여러 클래스와 관계를 맺는 경우에는 독립적으로 선언하는 것이 좋지만, 특정 클래스만 관계를 맺을 경우에는 중첩 클래스로 선언하는 것이 유지보수에 좋음
<br>
중첩 클래스는 선언 위치에 따라 멤버 클래스와 로컬 클래스로 나뉘고, 멤버 클래스는 static으로 선언했느냐에 따라 인스턴스 멤버 클래스와 정적 멤버 클래스로 나뉨

<br>
<br>
멤버 클래스 : 클래스의 멤버로서 선언되는 중첩 클래스
<br>
인스턴스 멤버 클래스 : A 객체를 생성해야만 B 객체를 생성할 수 있음
`class A {
	class B {..}
}`

<br>
정적 멤버 클래스 : A 객체를 생성하지 않아도 B 객체를 생성할 수 있음
`class A {
	static class B {..}
}`

<br>
<br>
로컬 클래스 : 메소드 내부에서 선언되는 중첩 클래스
<br>method가 실행할 때만 B 객체를 생성할 수 있음
`class A {
	void method() {
		class B{..}
	}
}`
